Contributor Code of Conduct
Clair is a disciplined cognitive AI system built on structure, verification, and honesty.
To maintain a healthy and productive environment, all contributors and participants are expected to follow this Code of Conduct.

This document is adapted from the Contributor Covenant, a widely used standard for open‑source communities.

🧭 Our Pledge
We pledge to make participation in Clair’s development a harassment‑free experience for everyone, regardless of:

experience level

technical background

personal identity

communication style

Clair is a project focused on engineering, research, and structured reasoning.
Respectful, professional behavior is required at all times.

🧠 Our Standards
Examples of behavior that contributes to a positive environment:
Using clear, concise communication

Respecting the project’s philosophy and design principles

Offering constructive feedback

Focusing on technical issues, not personal opinions

Following the contribution guidelines

Examples of unacceptable behavior:
Harassment, insults, or personal attacks

Dismissing or belittling others

Attempting to derail discussions

Ignoring project maintainers’ decisions

Trying to force design changes that violate Clair’s philosophy

🛡️ Enforcement Responsibilities
Project maintainers have the right and responsibility to:

remove or edit comments

reject contributions

ban contributors who violate this Code

enforce boundaries without debate

Maintainers’ decisions are final.

🚫 Scope
This Code of Conduct applies to:

GitHub issues

pull requests

discussions

project communication

any interaction related to Clair

🧾 Enforcement
Instances of abusive or unacceptable behavior may be reported by opening an issue or contacting the project maintainer directly.

All complaints will be reviewed and addressed appropriately.
